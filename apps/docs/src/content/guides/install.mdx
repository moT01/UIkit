The UIKit ships as six independent packages so each surface stays
honest. Tree-shake what you don't use; nothing pulls in the others.

## Per-package matrix

| Package                        | When to install                                |
| ------------------------------ | ---------------------------------------------- |
| `@freecodecamp/uikit`          | React component layer                          |
| `@freecodecamp/uikit-css`      | Tokens + BEM CSS, framework-agnostic           |
| `@freecodecamp/uikit-icons`    | SVG sprite + React icon helper                 |
| `@freecodecamp/uikit-js`       | Vanilla-JS adapters (modal, dropdown, tabs, …) |
| `@freecodecamp/uikit-tailwind` | Tailwind preset that mirrors every token       |
| `@freecodecamp/uikit-cdn`      | `<script>` + `<link>` tag bundle               |

## React

```bash
pnpm add @freecodecamp/uikit @freecodecamp/uikit-css
```

```tsx
import '@freecodecamp/uikit-css/index.css';
import { Button } from '@freecodecamp/uikit';

export default function Home() {
  return <Button variant='cta'>Start</Button>;
}
```

## Vanilla HTML

```bash
pnpm add @freecodecamp/uikit-css @freecodecamp/uikit-js
```

```html
<link
  rel="stylesheet"
  href="/node_modules/@freecodecamp/uikit-css/src/index.css"
/>
<button class="btn btn--cta">Start</button>
<script
  src="/node_modules/@freecodecamp/uikit-js/dist/uikit.global.js"
  defer
></script>
```

The `uikit-js` IIFE wires modals, dropdowns, tabs, toggle buttons, and
tooltips by data-attributes — no React required.

## Tailwind

```bash
pnpm add @freecodecamp/uikit-tailwind @freecodecamp/uikit-css
```

```js
// tailwind.config.js
import fcc from '@freecodecamp/uikit-tailwind';

export default {
  presets: [fcc],
  content: ['./src/**/*.{ts,tsx,html}']
};
```

The preset mirrors every CSS custom property as a Tailwind token
(`bg-cta-background`, `text-fg-primary`, etc.). Don't double-import
`uikit-css` if you also import `@tailwind base`.

## CDN

For prototypes and learner pages, the CDN bundle ships everything
through plain `<link>` and `<script>` tags. See the
[CDN guide](/guides/cdn) for tag layout and sprite mounting.
