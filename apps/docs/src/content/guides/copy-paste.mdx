Every `<PlaygroundCard>` on the playground (`/`) ships a copyable
HTML tab. This guide collects the recipes that span more than one
card — composition patterns the per-component HTML can't show alone.

## Form group with help block

```html
<div class="form-group">
  <label class="form-label" for="email">Email address</label>
  <input
    class="input"
    id="email"
    type="email"
    placeholder="camper@example.com"
    aria-describedby="email-help"
  />
  <p class="form-help" id="email-help">
    We send one curriculum update per week.
  </p>
</div>
```

## Modal triggered from a button

```html
<button class="btn btn--danger" data-modal-trigger="modal-reset">
  Reset progress
</button>

<div
  class="modal"
  id="modal-reset"
  role="dialog"
  aria-modal="true"
  aria-labelledby="modal-reset-title"
  data-open="false"
>
  <div class="modal__panel">
    <header class="modal__header">
      <p class="modal__title" id="modal-reset-title">Reset progress?</p>
      <button class="close-btn" data-modal-close aria-label="Close">×</button>
    </header>
    <div class="modal__body">
      <p>You will lose 17 of 32 saved steps.</p>
    </div>
    <footer class="modal__footer">
      <button class="btn" data-modal-close>Cancel</button>
      <button class="btn btn--danger" data-modal-close>Confirm</button>
    </footer>
  </div>
</div>
```

The `data-modal-trigger` + `data-modal-close` attributes are wired by
`uikit-js`. No JavaScript on your side.

## Card with progress bar

```html
<article class="card">
  <header class="card__header">
    <span class="card__dot card__dot--purple"></span>
    <p class="card__hours">300 HOURS</p>
  </header>
  <h3 class="card__title">Responsive Web Design</h3>
  <p class="card__body">Build five certification projects.</p>
  <div class="progress">
    <div class="progress__bar progress__bar--purple" style="width:62%"></div>
  </div>
  <footer class="card__footer">
    <span>62% complete</span>
    <a class="fcc-link" href="/learn/rwd">Resume →</a>
  </footer>
</article>
```
