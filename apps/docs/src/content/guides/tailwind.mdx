The Tailwind preset is a thin layer that translates every UIKit token
into a Tailwind utility prefix. `bg-cta-background` reads the same CSS
custom property the BEM classes do, so palette switches stay
synchronized across both surfaces.

## Install

```bash
pnpm add tailwindcss @freecodecamp/uikit-tailwind @freecodecamp/uikit-css
```

## Wire it up

```js
// tailwind.config.js
import fcc from '@freecodecamp/uikit-tailwind';

export default {
  presets: [fcc],
  content: ['./src/**/*.{ts,tsx,astro,html}'],
  // The preset already configures fontFamily, colors, spacing, and
  // borderRadius. Extend if you need project-only tokens.
  theme: {
    extend: {}
  }
};
```

```css
/* src/index.css */
@import 'tailwindcss';
@import '@freecodecamp/uikit-css';
```

## The double-import gotcha

Don't `@import 'tailwindcss'` AND import `@freecodecamp/uikit-css/index.css`
both, then layer your own utility-mode CSS on top. Pick one of these
two patterns:

1. **Tailwind utility-first** — import `tailwindcss` only; let the
   preset surface the tokens. Skip `uikit-css` entirely.
2. **BEM-first + Tailwind escape hatch** — import both, but order
   `uikit-css` before `tailwindcss` so utility classes win the
   cascade.

The shipped docs site uses (2) so the playground keeps its BEM look
while the Tailwind tab in each `<PlaygroundCard>` shows utility-only
markup.

## Palette switching

The preset reads CSS custom properties at runtime, so swapping
`<html class="dark-palette">` for `light-palette` instantly recolors
every `bg-*`, `text-*`, and `border-*` utility. No build step.

```html
<html class="dark-palette">
  …
</html>
<html class="light-palette">
  …
</html>
```
