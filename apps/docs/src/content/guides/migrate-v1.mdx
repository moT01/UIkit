The v1.0.0 GA release stabilises every component. The breaking
changes are short, opinionated, and concentrated in the form layer.

## Component renames

| Old                   | New                     |
| --------------------- | ----------------------- |
| `<Input.Helper>`      | `<HelpBlock>`           |
| `<Input.Label>`       | plain `<label htmlFor>` |
| `<FormControl.Group>` | `<FormGroup>`           |

## Prop renames

| Component  | Old              | New                  |
| ---------- | ---------------- | -------------------- |
| `<Button>` | `kind="cta"`     | `variant="cta"`      |
| `<Alert>`  | `tone="success"` | `variant="success"`  |
| `<Card>`   | `bordered`       | `variant="bordered"` |

## CSS class renames

| Old              | New                     |
| ---------------- | ----------------------- |
| `.btn-cta`       | `.btn.btn--cta`         |
| `.alert-success` | `.alert.alert--success` |
| `.card-title`    | `.card__title`          |

The double-underscore (`__`) and double-dash (`--`) BEM convention is
now consistent across every component.

## Token renames

| Old                   | New                    |
| --------------------- | ---------------------- |
| `--accent-primary`    | `--cta-background`     |
| `--accent-foreground` | `--cta-foreground`     |
| `--text-primary`      | `--foreground-primary` |

## Removed components

- `<Splash>` — fold into `<EmptyState>`.
- `<Banner>` — fold into `<Alert>` or `<Callout>`.

## codemod

A jscodeshift transform ships in
`@freecodecamp/uikit/codemods/v1.0.cjs`:

```bash
pnpm dlx jscodeshift \
  --transform=node_modules/@freecodecamp/uikit/codemods/v1.0.cjs \
  src
```

The transform handles every prop and component rename automatically.
CSS class renames need a manual find-and-replace; the table above is
exhaustive.
