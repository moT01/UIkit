`<ToggleButton>` is a pressable button that surfaces its state through
`aria-pressed`. Use it for icon-bar toolbars (bold / italic / strike)
and settings where the label never changes but the meaning flips.

## Keyboard

| Key           | Action                                     |
| ------------- | ------------------------------------------ |
| Space / Enter | Toggles the pressed state.                 |
| Tab           | Moves focus to the next focusable element. |

## Accessibility

Renders a native `<button>` with `aria-pressed="true|false"`. Screen
readers announce the state as "toggle button, pressed" /
"…not pressed". Label text should describe the action, not the state.

## Do & Don't

<DoDont>
  <Fragment slot='do'>
    <ul>
      <li>Keep the visible label stable — let `aria-pressed` carry the state.</li>
      <li>Group related toggles in a `role="toolbar"` container.</li>
      <li>Pair with a tooltip when the icon isn't self-explanatory.</li>
    </ul>
  </Fragment>
  <Fragment slot='dont'>
    <ul>
      <li>Don't swap the label between "On" and "Off" — use `aria-pressed` instead.</li>
      <li>Don't use it for one-off actions; reach for `<Button>` there.</li>
      <li>Don't nest a ToggleButton inside a form submit — it dispatches clicks.</li>
    </ul>
  </Fragment>
</DoDont>
