AuthLayout is the centered-card shell for login, signup, forgot-password,
and magic-link flows. Drop your form inside as children; add a brand
lockup above and a legal footer below via the `brand` and `footer`
slots. Enable `pattern` to turn the background into a muted grid
texture for a little visual interest on otherwise-empty auth routes.

## Accessibility

The card wrapper is a native `<main>` — nothing fancier needed.
Ensure the form inside carries an accessible heading (`<h1>` like
"Sign in") so screen reader users know the purpose of the route.

## Do & Don't

<DoDont>
  <Fragment slot='do'>
    <ul>
      <li>
        Use AuthLayout for every unauthenticated route — login, register,
        forgot, magic-link.
      </li>
      <li>
        Pair with Fieldset + FormControl to keep each form group semantically
        grouped.
      </li>
      <li>
        Reach for `pattern` on otherwise-empty pages; drop it when the form body
        is dense.
      </li>
    </ul>
  </Fragment>
  <Fragment slot='dont'>
    <ul>
      <li>
        Don't use AuthLayout for authenticated pages — SidebarLayout or
        StackedLayout fit better.
      </li>
      <li>
        Don't stuff navigation chrome into `brand`; keep it purely visual.
      </li>
      <li>Don't nest two AuthLayouts — one shell per route.</li>
    </ul>
  </Fragment>
</DoDont>
