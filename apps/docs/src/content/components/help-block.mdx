`<HelpBlock>` is the tiny explanatory line that sits under a control.
The `variant` prop flips tone for success or error without changing
DOM shape — so `aria-describedby` wiring stays stable across states.

## Props

<PropTable
  rows={[
    {
      name: 'variant',
      type: "'default' | 'success' | 'error'",
      default: "'default'",
      description:
        'Colour tone. `error` also sets `role="alert"` so the update is announced.'
    },
    {
      name: 'children',
      type: 'ReactNode',
      description: 'The microcopy.'
    }
  ]}
/>

## Accessibility

Consumes a stable id from its parent `<FormControl>` (via context) and
joins that control's `aria-describedby` chain. When `variant="error"`,
the block adds `role="alert"` so screen readers announce the change
without a manual re-focus.

## Do & Don't

<DoDont>
  <Fragment slot='do'>
    <ul>
      <li>Write microcopy that states the rule: "8+ characters, 1 number."</li>
      <li>Use `variant="error"` only when the control is actually invalid.</li>
      <li>Keep to one line; wrap long guidance in a dedicated `<Callout>`.</li>
    </ul>
  </Fragment>
  <Fragment slot='dont'>
    <ul>
      <li>Don't use HelpBlock for multi-sentence guidance — reach for a Callout.</li>
      <li>Don't hide error messages; they must stay visible until resolved.</li>
      <li>Don't use red text without the `error` variant — accessibility chain depends on it.</li>
    </ul>
  </Fragment>
</DoDont>
