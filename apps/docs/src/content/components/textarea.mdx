Textarea is the native multi-line input, restyled to match the rest of
the terminal form shell. Drop in `variant="mono"` for code and command
capture; flip `autoResize` on when the control should grow and shrink
with its content instead of using the default vertical scroll affordance.

## Accessibility

Native `<textarea>` — every screen reader and keyboard affordance works
out of the box. Pair with a `<label>` (or FormGroup + HelpBlock) for
the accessible name. When `invalid` is set, surface a matching error
message through HelpBlock so the assistive tech announcement has
something to read.

## Do & Don't

<DoDont>
  <Fragment slot='do'>
    <ul>
      <li>
        Set a sensible default `rows` so the control has a reasonable starting
        footprint.
      </li>
      <li>
        Use `autoResize` for comment boxes and prompts — the overflow guesswork
        disappears.
      </li>
      <li>
        Reach for `variant="mono"` whenever the user is entering code or shell
        commands.
      </li>
    </ul>
  </Fragment>
  <Fragment slot='dont'>
    <ul>
      <li>Don't forget the label — every Textarea needs an accessible name.</li>
      <li>
        Don't toggle `autoResize` on controls that must honour a fixed height.
      </li>
      <li>Don't reach for Textarea when a single-line Input will do.</li>
    </ul>
  </Fragment>
</DoDont>
