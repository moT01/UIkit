`<Tooltip>` wraps any trigger with a short explanatory hint. The bubble
renders inline next to the trigger so there is no portal or positioning
script — keep the hint under a dozen words.

## Usage

```tsx
import { Tooltip } from '@freecodecamp/uikit/overlays';

<Tooltip content='Copy link'>
  <button aria-label='Copy' className='btn btn--ghost'>
    🔗
  </button>
</Tooltip>;
```

## Accessibility

The bubble carries `role='tooltip'`; the trigger wrapper has `tabIndex={0}`
so keyboard users can focus it and see the hint. Tooltips are not a
substitute for a visible label: if the trigger has no text, give it an
`aria-label`.

## Do & Don't

<DoDont>
  <Fragment slot='do'>
    <ul>
      <li>Keep tooltip copy under a dozen words.</li>
      <li>Reserve tooltips for non-essential hints; essential labels stay visible.</li>
      <li>Combine with `aria-label` on icon-only triggers.</li>
    </ul>
  </Fragment>
  <Fragment slot='dont'>
    <ul>
      <li>Don't stuff interactive content inside the tooltip — use `<Dropdown>` or `<Modal>`.</li>
      <li>Don't rely on a tooltip to convey required information.</li>
      <li>Don't nest Tooltips.</li>
    </ul>
  </Fragment>
</DoDont>
