`<FormControl>` wires the label ↔ control ↔ help relationship so
consumers don't hand-generate ids. Drop an `<Input>`, `<Textarea>`, or
render prop inside and the a11y plumbing flows automatically.

## Accessibility

Generates deterministic ids via `useId()` and wires `htmlFor` on the
label plus `aria-describedby` on the control. When `error` is present,
the control also gains `aria-invalid="true"`. Screen readers announce
the label, the current value, and the help/error text in order.

## Do & Don't

<DoDont>
  <Fragment slot='do'>
    <ul>
      <li>Place one FormControl per field; wrap the whole group in `<FormGroup>`.</li>
      <li>Use `help` for always-visible guidance, `error` for validation failures.</li>
      <li>Keep labels concise — they're announced before every re-read.</li>
    </ul>
  </Fragment>
  <Fragment slot='dont'>
    <ul>
      <li>Don't wire `id`/`htmlFor` manually — FormControl owns that contract.</li>
      <li>Don't hide the label; use `sr-only` only when layout truly demands it.</li>
      <li>Don't stack help _and_ error for the same field — error wins.</li>
    </ul>
  </Fragment>
</DoDont>
