`<FormGroup>` is the spacing wrapper for one form field. It owns the
vertical rhythm between a label, control, and help/error block so
consumers don't fight margins.

## Accessibility

Adds no ARIA on its own — rely on nested `<FormControl>` for
`label ↔ input` wiring, and `<HelpBlock>` for descriptive text
(`aria-describedby`). When grouping multiple related inputs, use
`as="fieldset"` and include a `<legend>` child.

## Do & Don't

<DoDont>
  <Fragment slot='do'>
    <ul>
      <li>Wrap a single control + label + help text in one group.</li>
      <li>Reach for `as="fieldset"` when the group spans multiple inputs (radio set).</li>
      <li>Place `<HelpBlock>` _after_ the control so it reads naturally.</li>
    </ul>
  </Fragment>
  <Fragment slot='dont'>
    <ul>
      <li>Don't stack raw margin between field rows — that's FormGroup's job.</li>
      <li>Don't nest FormGroups; use one per logical field.</li>
      <li>Don't skip the `<legend>` when `as="fieldset"` — screen readers need it.</li>
    </ul>
  </Fragment>
</DoDont>
