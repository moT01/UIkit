`<Dropdown>` is the menu-button compound: a trigger (`Dropdown.Toggle`),
a panel (`Dropdown.Menu`), and items (`Dropdown.Item`). It owns
keyboard navigation and click-outside dismiss so consumers stay
declarative.

## Keyboard

| Key                | Action                                            |
| ------------------ | ------------------------------------------------- |
| Space / Enter      | Toggle the menu (on Toggle) / activate (on Item). |
| Escape             | Close the menu and return focus to the Toggle.    |
| Outside mouse-down | Closes the menu.                                  |

## Accessibility

Follows the APG [menu button](https://www.w3.org/WAI/ARIA/apg/patterns/menu-button/)
pattern. The toggle sets `aria-haspopup="menu"`, `aria-expanded`, and
`aria-controls`; items use `role="menuitem"`. Active items expose
`aria-current="true"`.

## Do & Don't

<DoDont>
  <Fragment slot='do'>
    <ul>
      <li>Use for low-frequency actions that would crowd the surface otherwise.</li>
      <li>Place an `active` marker on the "current" item when it reflects state.</li>
      <li>Keep item count under ~7 — past that, reach for `<Combobox>` or `<CommandPalette>`.</li>
    </ul>
  </Fragment>
  <Fragment slot='dont'>
    <ul>
      <li>Don't use it for navigation menus — `<Navbar>` owns that pattern.</li>
      <li>Don't nest dropdowns; reach for `<CommandPalette>` for multi-level menus.</li>
      <li>Don't place primary destructive actions inside; promote them to a dialog.</li>
    </ul>
  </Fragment>
</DoDont>
