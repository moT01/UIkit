Badges surface small pieces of structured metadata next to a label — a
status chip, a count, or a category tag. The default variant is the neutral
tag; the tone variants map to the semantic foreground tokens.

## Usage

```tsx
import { Badge } from '@freecodecamp/uikit/primitives';

<Badge variant='success'>Stable</Badge>;
```

## Accessibility

Badges are decorative by default — they render as `<span>` with no role.
If a badge carries meaning that is not repeated in adjacent text, add an
`aria-label` so screen readers announce it. Avoid using colour alone to
communicate status.

## Do & Don't

<DoDont>
  <Fragment slot='do'>
    <ul>
      <li>Keep badges to one or two words.</li>
      <li>
        Repeat the meaning in text for screen-reader users when the badge
        carries status.
      </li>
      <li>
        Use `success` / `warning` / `danger` sparingly — reserve them for real
        signal.
      </li>
    </ul>
  </Fragment>
  <Fragment slot='dont'>
    <ul>
      <li>Don't use Badge as a button — it is not interactive.</li>
      <li>Don't line up more than three badges in a row.</li>
      <li>Don't colour a badge purely for decoration.</li>
    </ul>
  </Fragment>
</DoDont>
