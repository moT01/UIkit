Pick `level` for the semantic outline, pick `size` for the visual weight.
Decoupling lets you keep the page's H1→H6 structure intact while still
compressing a subheading into something discreet.

## Props

<PropTable
  rows={[
    {
      name: 'level',
      type: '1 | 2 | 3 | 4 | 5 | 6',
      default: '2',
      description:
        'Rendered heading tag. Invalid values clamp to the nearest h1/h6.'
    },
    {
      name: 'size',
      type: "'display' | 'xl' | 'lg' | 'md' | 'sm'",
      default: "'md'",
      description: 'Visual size. Independent of level.'
    }
  ]}
/>

## Accessibility

Always use `level` to match the document outline. Size is a presentation
choice; a `<h6 size="display">` is legitimate if the outline requires it.

## Do & Don't

<DoDont>
  <Fragment slot='do'>
    <ul>
      <li>Match `level` to the outline, not the visual target.</li>
      <li>Use `size="display"` once per page at most.</li>
    </ul>
  </Fragment>
  <Fragment slot='dont'>
    <ul>
      <li>
        Don't use Heading for marketing titles with custom fonts — that's
        content, not system.
      </li>
      <li>Don't skip heading levels for visual spacing.</li>
    </ul>
  </Fragment>
</DoDont>
