Alerts broadcast state changes the user must notice but not always act on.
They render with `role="alert"` so assistive tech announces the content the
moment it appears.

## Usage

```tsx
import { Alert } from '@freecodecamp/uikit/data-display';

<Alert variant='success' title='Saved'>
  Your progress is synced across every device.
</Alert>;
```

## Accessibility

Renders a `div` with `role="alert"` — assistive tech announces the content
immediately. Keep the message concise: screen readers interrupt the user as
soon as the node mounts. For non-urgent feedback reach for `<Callout>` or
`<Toast>` instead.

## Do & Don't

<DoDont>
  <Fragment slot='do'>
    <ul>
      <li>Write alerts in the active voice — "We saved your changes."</li>
      <li>Pair the `danger` variant with one recovery action.</li>
      <li>
        Keep alerts short. Link out if the explanation is more than two lines.
      </li>
    </ul>
  </Fragment>
  <Fragment slot='dont'>
    <ul>
      <li>Don't stack more than one `danger` alert on a page.</li>
      <li>Don't use Alert for marketing copy — use `<Callout>`.</li>
      <li>Don't swap Alert and Toast: Toast is transient, Alert is persistent.</li>
    </ul>
  </Fragment>
</DoDont>
