Fieldset is the grouping container for related form controls. It renders
a native `<fieldset>` with an optional `<legend>`, carrying our border
and spacing tokens so the group reads as a single terminal block. Pair
it with FormGroup, FormControl, Checkbox, Radio, and friends.

## Accessibility

Uses the native `fieldset` + `legend` pair — screen readers announce the
legend as the group label for every control inside. Prefer this over an
ad-hoc `div` + heading pattern whenever a group of controls needs a
shared name. `disabled` cascades automatically; no extra ARIA needed.

## Do & Don't

<DoDont>
  <Fragment slot='do'>
    <ul>
      <li>
        Group related controls (address fields, preference toggles) inside a
        single Fieldset.
      </li>
      <li>
        Give every Fieldset a `legend` — it's the group's accessible name.
      </li>
      <li>
        Use `disabled` at the Fieldset level to lock a whole section at once.
      </li>
    </ul>
  </Fragment>
  <Fragment slot='dont'>
    <ul>
      <li>
        Don't nest Fieldsets more than one level — screen readers get noisy
        fast.
      </li>
      <li>Don't reach for Fieldset as a generic card — use Card or Panel.</li>
      <li>
        Don't hide the legend visually; if the group truly needs no label, it
        probably doesn't need a Fieldset.
      </li>
    </ul>
  </Fragment>
</DoDont>
